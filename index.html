High Performance Computing + Computer Vision

Parallel Crowd Counting from Video using YOLOv8

A High Performance Computing project that performs crowd detection and counting from video using YOLOv8, with a direct CPU vs GPU performance benchmark.

The pipeline extracts video frames, runs person detection with deep learning, and demonstrates how CUDA-based GPU acceleration drastically reduces processing time for real-world surveillance workloads.

Project Author: Godishala Kruthik Roshan

GitHub Repository
Use Cases and Scope
  1. Smart surveillance and crowd density monitoring from video streams.
  2. Public event monitoring and safety planning support.
  3. Traffic movement analysis and urban mobility planning.
  4. Disaster management and emergency crowd response scenarios.
  5. Urban planning insights from large-scale footfall data.
System Architecture
  1. Video Input Module accepts crowd footage for processing.
  2. Frame Extraction Module uses OpenCV for frame-level processing.
  3. CPU Execution Module runs YOLOv8 sequentially on CPU.
  4. GPU Execution Module runs YOLOv8 with CUDA acceleration on GPU.
  5. Performance Analysis Module measures time, speedup, and efficiency.
  6. Visualization Module presents CPU vs GPU benchmark graphs.
Performance and Output
  1. CPU execution around 75 seconds for benchmark workload.
  2. GPU execution around 4 seconds with parallel acceleration.
  3. Observed speedup near 16x using CUDA-enabled GPU execution.
  4. Generated outputs include execution comparison and speedup graphs.
  5. Demonstrates practical advantage of HPC methods in video analytics.

Technology Stack

  • Language: Python.
  • Model: YOLOv8 (Ultralytics).
  • Video Processing: OpenCV.
  • Deep Learning: PyTorch.
  • Acceleration: CUDA on GPU.
  • Visualization: Matplotlib.

Workflow and Run

  • Clone repository and install required dependencies.
  • Run crowd_counting.ipynb in Colab or Jupyter Notebook.
  • Process input video with YOLOv8 person detection.
  • Compare CPU and GPU timings and visualize benchmark results.
